# Documentation Archive

**Status:** archival

This directory contains documents that are still useful for historical context, implementation archaeology, or future redesign work, but should not be part of the main reading path for understanding the product today.

## What belongs here

Typical archive candidates:
- completed roadmap phase detail
- migration plans after the migration is largely complete
- convergence or retirement plans after the convergence work has landed
- implementation checklists for shipped work
- exploratory or transitional planning docs that are no longer canonical

## How to use the archive

Use archived docs when you need to answer questions like:
- how did the redesign converge onto the current workflow?
- why was a given legacy path retired?
- what assumptions shaped earlier implementation phases?
- what historical detail was intentionally removed from the active roadmap?

Do **not** use archived docs as the main source of truth for:
- current product behavior
- current roadmap priorities
- current operator workflow
- current payload contracts

For those, start with:
- `../docs-index.md`
- `../product-thesis.md`
- `../features-and-capabilities.md`
- `../recommendation-methodology.md`
- `../architecture.md`
- `../roadmap.md`
